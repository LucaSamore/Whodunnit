# Sprint 3 (20/10/2025 - 26/10/2025)

## Goal

The third sprint aims to advance both the game's core logic and the user interface toward a more complete and functional state.
The primary objective is to finalize the case generation implementation, which was deferred from Sprint 1, and to integrate it with a graphical representation of the knowledge graph.
On the UI side, we will complete the clues management interface and refine the main game page and blackboard components to enhance the overall user experience.
Additionally, we will establish the Continuous Deployment pipeline to streamline the release process.

## Review
The case generation feature, originally planned for Sprint 1, has been successfully completed during this sprint.
The implementation of the logic for generating mystery cases was finalized, along with a visual representation of the knowledge graph to aid in debugging and understanding the case structure.
On the UI front, significant progress has been made on the game interface components. The clues management system has been refined, and both the main game page and the game blackboard have been enhanced with improved layouts and functionality.
The Continuous Deployment pipeline has been successfully configured, enabling automated deployment of the application and streamlining the release workflow.

## Retrospective

This sprint demonstrated improved team coordination and availability. The contribution to the case generation was crucial, and having him also work on the graph visualization created a stronger connection between the model and its visual representation.
The parallel advancement of both core logic and UI components is yielding a more cohesive product. The team's familiarity with ScalaFX, gained in previous sprints, has notably accelerated the UI development work.
Moving forward, the integration between the case generation system and the user interface will be a key focus, ensuring that the generated cases are properly displayed and interactive within the game screens.
